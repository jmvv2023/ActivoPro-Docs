Guía de Usuario ActivoPro®

Bienvenido a la documentación oficial. Esta es una guía detalladas para el uso de las funcionalidades de ActivoPro®.

1. Contactos (Terceros)

El módulo de Contactos es la base del sistema, permitiendo dar de alta, modificar y gestionar clientes, proveedores, acreedores y cuentas bancarias. Estos elementos o terceros con los que interactua su operativa son cruciales para el correcto funcionamiento de las operaciones de contabilidad, facturación y la tesorería.

1.1 Clientes/Usuarios – Altas, Modificaciones y Consultas

Para gestionar la ficha de un Cliente, siga la siguiente ruta: Menú → Contactos → Clientes/Usuarios.

ALTA RÁPIDA

Para dar de ALTA un nuevo cliente, haga clic en el botón Nuevo. Por defecto el sistema le propondrá un identificador automático numérico, puede aceptarlo o inreoducir manuelamente el identificador deseado. Los campos obligatorios los encontrará identificados en la pestaña General.

Pestañas de la Ficha del Cliente:

  • General: Datos básicos (nombre, NIF/CIF). Para particulares marque “PARTICULAR” en Clase de Cliente.
  • Condiciones de Cobro: Define la forma de cobro. (Ver sección 3 Tesorería).
  • Cuentas: Cuenta de ingreso y el IVA por defecto.
  • Información Adicional: Contactos internos y notas.
  • FACe: Centros de facturación para FACTURAe.
  • Facturas: Histórico de facturas emitidas.

1.2 Proveedores/Acreedores – Altas y Gestión

Para gestionar Proveedores o Acreedores, utilice Menú → Contactos → Proveedores o → Acreedores.

GESTIÓN DE DATOS CRUCIALES

Las cuentas de Proveedores y Acreedores deben crearse aquí para garantizar que la información fiscal y la forma de pago queden correctamente registradas, vinculándose al Plan Contable.

Pestañas Específicas:

  • Condiciones de Pago: Seleccione la forma de pago acordada.
  • Cuentas: Cuenta de compra/gasto y el IVA por defecto.
  • Información Adicional: Contactos y notas.

4. Apuntes y Asientos

4.1 ALTA ASIENTOS FACTURAS RECIBIDAS

Para dar de alta un nuevo asiento de facturas recibidas, navegue a: Menú > Asientos > Facturas Recibidas > Compras, Gastos y Servicios. Cumplimente la información requerida y haga clic en Guardar para confirmar el alta.

AUTOMATIZACIÓN

Al configurar la cuenta del proveedor/acreedor, el sistema sugerirá automáticamente la cuenta de gasto asociada y el porcentaje de IVA/IRPF si corresponde, basándose en la ficha del tercero.

Campos Clave del Asiento:

Fecha operación

Fecha de registro contable. Puede coincidir o no con la fecha de la factura original.

Proveedor/Acreedor

Indique el nombre o cuenta contable del proveedor/acreedor.

Concepto

Seleccione de la lista el concepto que mejor represente el tipo de movimiento contable a realizar.

No / De fecha

No: Número de factura del proveedor. De fecha: Fecha de factura del proveedor.

Serie/Documento

Indique la serie/documento contable. Si hay numeración automática, el sistema la asignará.

Total factura

Total con IVA incluido.

% IVA / Base Imponible / Cuota

Introduzca el % de IVA y la base imponible. El sistema calculará la cuota; si hay diferencias de redondeo se puede ajustar manualmente.

Deducibilidad

Seleccione: Deducible, Exento, Parcial, etc.

IRPF Base / IRPF % / IRPF Cuota

Indique la base y el tipo de retención para calcular la cuota.

4.2 ALTA ASIENTOS BANCOS

Menú → Asientos → Bancos, Contabilizar Apuntes. Complete y Guardar.

Campos Clave y Contabilización:

Fecha operación

Fecha del movimiento bancario.

Cuenta contable

Ej. 572.XXX

Contrapartida

Ej. 430 clientes, 410 proveedores.

Cobros / Pagos

Indique importe en Cobros (DEBE) o Pagos (HABER).

Comisión / Cuenta de Gasto

Si existe comisión, especifique cuenta de gasto.

Asiento con Comisión

La comisión se carga al DEBE y el saldo al HABER en la cuenta bancaria.

4.3 Cartera Cobros y Pagos (Previsión)

La contabilización de Cobros y Pagos se enlaza con la previsión del sistema.

Tras pulsar Guardar (F2), el programa mostrará facturas pendientes para seleccionar y aplicar.

LA REGLA DE ORO

El total del asiento debe coincidir con el total de las facturas seleccionadas; si no coincide, el sistema aplicará lógica de diferencias.

Escenarios de Diferencia en la Aplicación:

  1. a) Importe Inferior: Se aplica a la última factura seleccionada, quedando pendiente la diferencia.
  2. b) Importe Superior: El sobrante queda como cobro/pago a cuenta de próximas facturas.
  3. c) No aplicable o sin facturas: Queda como pendiente de aplicar.

ADJUNTAR DOCUMENTACIÓN

Puede escanear y adjuntar comprobantes directamente al asiento.

Tipos de Asientos para Cobros y Pagos:

Asiento de Caja (Efectivo)

Ruta: Menú → Asientos → Caja

Asiento de Bancos (Cheques, Transf.)

Ruta: Menú → Asientos → Bancos, Contabilizar apuntes

Asiento de Varios (Tarjetas, Efectos)

Ruta: Menú → Asientos → Varios → Varios

Gestión de Importes Pendientes de Aplicar:

Para aplicar importes pendientes use:

  • Cobros pendientes: Menú → Tesorería → Cobros → Importes pdtes de aplicar
  • Pagos pendientes: Menú → Tesorería → Pagos → Importes pdtes de aplicar

4.4 Norma 43 y CBTF/EXCEL (Importación de Movimientos Bancarios)

Permite automatizar la creación de asientos a partir de ficheros de movimientos bancarios (.txt o Excel).

Paso 1: Seleccionar Fichero y Procesar

Menú → Asientos → Importaciones asientos Excel → N43 o CBTF/EXCEL. Seleccione archivo y pulse Importar.

ACTIVO·PRO® generará una plantilla con los conceptos y abrirá una hoja Excel con importes.

PREREQUISITO

La cuenta bancaria debe estar dada de alta: Menú → Contactos → Bancos.

Paso 2: Mantenimiento de la Plantilla (Limpiar y Enseñar)

Use el botón Plantilla y seleccione la del banco para identificar líneas relevantes.

Reglas de Configuración:

  • Deje en las columnas Concepto y Operación la parte coincidente del texto.
  • Columna X: use NO para conservar la línea; SI para eliminarla.
  • Complete nº de cuenta, clave y centro para las líneas con X = NO.

Ejemplo de Limpieza de Conceptos:

CONCEPTOOPERACIONX
VENCIMENT :PIERSI
LLIURADOR : ENUMBERS ONLINE SONO
CONCEPTE :SI
REFERENCIA CLIENT :SI

En este ejemplo, solo la línea con LLIURADOR : ENUMBERS ONLINE SO se conservará.

Excepciones (Columna 'EX')

Use la columna EX si desea que un concepto corto no coincida dentro de conceptos más largos y apunte a cuentas diferentes.

Paso 3: Volver a Procesar el Fichero

Vuelva a procesar para verificar la plantilla; la hoja mostrará solo los conceptos con X = NO.

AHORRO DE TIEMPO

La limpieza y enseñanza se realiza una sola vez; en procesos posteriores solo aparecerán nuevos conceptos no reconocidos.

4.6 Inbox de Documentos

El Inbox de Documentos es una herramienta centralizada para subir, organizar y procesar documentos financieros y contables directamente en el sistema. Acceda a ella desde Menú → Asientos → Inbox. Esta funcionalidad optimiza el flujo de trabajo al permitir adjuntar documentos a asientos contables de manera intuitiva y eficiente, reduciendo errores y mejorando la trazabilidad.

BENEFICIOS UX

El Inbox está diseñado para ser intuitivo: arrastre y suelte documentos, o envíelos por email para un procesamiento rápido. La interfaz visualiza documentos en bandejas claras, facilitando la navegación en dispositivos móviles con toques simples.

Subir Documentos

Para subir documentos, haga clic en el botón Subir Doc. y seleccione archivos PDF desde su dispositivo (facturas, recibos, comprobantes de pago, etc.). Alternativamente, envíe documentos por correo electrónico a:

  • login.facturas@activopro.net: Para facturas recibidas (sustituya "login" por su ID de acceso a la empresa).
  • login.docs@activopro.net: Para otros documentos financieros.

Esta opción de email facilita la integración con escáneres o apps móviles, permitiendo un flujo de trabajo remoto y dinámico.

Organización en Bandejas

El Inbox se divide en dos categorías principales para una clasificación inicial clara y eficiente:

Facturas Recibidas

Suba aquí las facturas recibidas de proveedores. Posteriormente, puede moverlas a otras bandejas si es necesario.

Otros

Incluye recibos, comprobantes y cualquier documento financiero no clasificado como factura recibida.

Para mover documentos entre bandejas (Facturas Recibidas, Otros o Papelera):

  1. Seleccione el documento.
  2. Elija el nuevo destino en el menú Mover a.
  3. Haga clic en Guardar.

GESTIÓN DE ERRORES

Si sube un documento duplicado, erróneo o que desea posponer, muévalo a la Papelera. Los documentos en Papelera no se eliminan permanentemente, permitiendo recuperación si es necesario.

Clasificación en "Otros"

En la bandeja "Otros", los documentos se clasifican por tipo de programa contable para una contabilización precisa:

  • Caja: Para movimientos en efectivo.
  • Banco: Para transacciones bancarias.
  • Varios: Para otros tipos de asientos (tarjetas, efectos, etc.).

Para cambiar la clasificación:

  1. Seleccione el documento.
  2. En el campo Abrir con, elija el programa deseado (Caja, Banco o Varios).
  3. Haga clic en Guardar.

Esta clasificación dinámica asegura que el documento se abra en la pantalla de contabilización correcta, mejorando la usabilidad en flujos de trabajo repetitivos.

Proceso de Contabilización

Una vez clasificado el documento en la bandeja y programa adecuados:

  1. Seleccione el primer documento a procesar en la bandeja.
  2. Haga clic en Contabilizar (icono de lápiz).
  3. Se abrirá la pantalla de contabilización correspondiente (ej. Caja), con el documento visualizado a la izquierda para referencia fácil durante el ingreso de datos.
  4. Complete los campos del asiento y guarde.

Al finalizar:

  • El sistema genera el apunte contable con el documento adjunto automáticamente.
  • Avanza al siguiente documento del mismo tipo (ej. todos los de "Caja") para un procesamiento secuencial eficiente.
  • Si prefiere procesar individualmente, seleccione documentos uno a uno en lugar de seguir el flujo automático.

CONSEJO PARA DISPOSITIVOS MÓVILES

En móviles, la vista del documento se adapta automáticamente: toque para ampliar y comparar con los campos de ingreso. Esto asegura una experiencia fluida sin necesidad de zoom manual constante.

6. Plan Contable (Estructura)

La gestión contable es el pilar de la información financiera de cualquier entidad. ACTIVO·PRO® ofrece una solución flexible para configurar el Plan Contable.

6.1 Configuración Inicial

Al alta de una nueva entidad, ACTIVO·PRO® permite:

  1. Selección Estándar: Usar uno de los planes preconfigurados.
  2. Personalización: Tomar un estándar y adaptarlo.
  3. Importación: Importar la estructura actual de la empresa.

6.2 Estructura y Codificación del Código de Cuenta

Estructura jerárquica: Grupo + Cuenta = Mayor Contable.

A. El Grupo

Agrupa cuentas de la misma naturaleza.

  • Dígitos: 3 o 4.
  • Jerarquía: No puede haber grupos de 4 sin el grupo de 3 asociado.

B. La Cuenta

Detalle dentro del grupo.

  • Longitud máxima: 6 dígitos.

C. El Mayor Contable

Código final usado en apuntes (Grupo + Cuenta).

LONGITUD TOTAL

El Mayor contable tendrá una longitud total de 9 o 10 dígitos.

6.3 Cuentas de Terceros

Deben darse de alta desde Contactos para capturar datos fiscales y condiciones de pago.

Grupos de Terceros:

  • Proveedores
  • Acreedores
  • Clientes
  • Deudores
  • Entidades de crédito
  • Empleados

ACTIVO·PRO® gestiona auxiliares y vincula por CIF/NIF.

6.4 Automatización Tributaria (IVA, IGIC, IRPF)

La plataforma crea automáticamente cuentas tributarias según tipo impositivo.

Creación Automática de Cuentas:

Mediante Auxiliares → Tipos de IVA/IGIC/IRPF se asignan grupos; la cuenta se calcula automáticamente. Ejemplo (texto):

Cuenta Tributaria = Número de Grupo + (Número de Tipo Impositivo × 100)

Así, al indicar el tipo impositivo en un documento, la contabilización de la cuota es automática.
